# Онбординг — первый день в kp-generator

Привет. Этот документ проведёт тебя от «склонировал репо» до «понимаю, что и где». Рассчитан на ~3-4 часа.

Прежде чем читать дальше — пройдись по `README.md` в корне проекта (15 минут). Там общая картина: что делает сервис, тех-стек, ключевые файлы, правила работы. Дальнейший текст предполагает, что ты с этим знаком.

## Шаг 1. Environment (15 минут)

```bash
git clone <ssh-url>
cd kp-generator
npm install
cp .env.local.example .env.local
```

Получи Supabase URL + anon key (от Влада через 1Password / Signal — не email). Впиши в `.env.local`.

```bash
npm run dev
```

Проверка готовности: открой http://localhost:3000, нажми **«Синхронизировать из Google Sheets»** внизу формы. Должна появиться надпись «Загружено N позиций» (N ≈ 40-60 в зависимости от версии каталога). Если не подгрузилось — спроси Влада, возможно sheet перестроили.

## Шаг 2. Сгенерировать тестовое КП (20 минут)

Сделай два КП — по одному на каждый тип шаблона.

### Кейс А — inno QR (быстрый, без оборудования)

1. Компания: **ИННО**
2. Клиент: «Тест QR» (любое имя)
3. Локаций: 5
4. Лицензия: **inno QR**
5. Период подписки: **12 месяцев**
6. Услуги: оставь пустыми
7. Рассчитать КП → откроется превью.
8. Покрути по строкам — кликни на цену, поправь, нажми Enter. Сумма пересчитывается. Кликни на крестик в конце строки — должна появиться кнопка «Отменить» в нижней-центральной части на 6 секунд.
9. Скачать PPTX.

Открой `.pptx` в PowerPoint / Keynote. **Что должно быть:**
- Слайд 1-3 — обложка / преимущества / точки контакта (стандартный inno-блок)
- Слайд 4 — «Детализация стоимости» с одной строкой «inno QR × 5 лок. (12 месяцев)» в карточке «Лицензии и подписки»
- Слайд 5 — финальная заглушка
- Цена лицензии показана как **«8 000/мес»** (а не 96 000 целиком)
- Большой «К оплате» внизу: 96 000 ₽

### Кейс Б — inno Kiosk (полный, с оборудованием)

1. Компания: **ИННО**, Клиент: «Тест Kiosk», Локаций: 1
2. Лицензия: **inno Kiosk**, Устройств: 1, Крепление: настольный
3. Период: 12 месяцев
4. Услуги: внедрение + 50 позиций контента
5. Рассчитать → превью

Должно быть **8 позиций оборудования** (планшет, кронштейн, адаптер, БП, кабель, угловой переходник, хаб, крепление пинпада). Это ключевая проверка — раньше 8-я строка молча терялась в .pptx (см. `docs/audits/AUDIT_FULL_2026_05_14.md`, P0-1). Сейчас должна быть видна и в превью, и в .pptx.

Скачай → открой → убедись, что в карточке «Оборудование» все 8 строк присутствуют и сумма по ним совпадает с ИТОГО карточки (`83 300 ₽` для дефолтных цен).

### Кейс В — стресс-тест overflow (опционально)

1. Сгенерируй inno Kiosk PRO с моделью киоска и десятком галочек на доп.опции (ККТ, ФН, сканер и т.д.).
2. Кнопка «+ Добавить» в превью должна заблокироваться при `11/11` для оборудования, `3/3` для лицензий, `6/6` для услуг.
3. Если попробуешь скачать с overflow — должен прилететь blocking-alert «Невозможно выгрузить .pptx — превышены лимиты шаблона».

## Шаг 5. Тур по коду (1 час)

Открой `src/` в редакторе. Пройдись по файлам в этом порядке:

### `src/lib/calculator.ts` (~360 LOC)

Сердце расчётов. Прочти от начала до конца. Особое внимание:

- `interface LineItem` — `months` опционален, по умолчанию 1. Это ключевое для подписок (Phase 3 фикса).
- `recomputeLineTotal()` — **единственный источник истины** для формулы строки. Любая правка `total` должна идти через него.
- `calculateKP()` — главная функция. Разбита по веткам (kiosk, kiosk_pro, qr, ecomm, findir, bonda_bi). Каждая ветка делает push в массивы `equipItems` / `licItems` / `svcItems`.
- Внизу — `formatMoney`: Intl-форматирование в рублях для UI. **НЕ** используется в .pptx (там свой `fmtNum`).

### `src/lib/generatePptx.ts` (~700 LOC)

Самый «алгоритмически сложный» файл проекта. Тут происходит прямая XML-манипуляция. Прочти, но не пытайся осознать всё с первого раза.

Ключевые понятия:

- `CardMap` — описание одной карточки в шаблоне (контейнер, заголовок, строки, разделители, ИТОГО). Три карточки: LEFT (Оборудование), RIGHT_TOP (Лицензии), RIGHT_BOTTOM (Услуги).
- `PPTX_TEMPLATE_LIMITS` — потолки: 11 / 3 / 6 после клонирования.
- `expandCardRows()` — клонирует row-shape с новыми именами (`Text Ext 99_0..4`) и уникальными `<p:cNvPr id>` (через `findMaxCNvId`), сдвигая Y на нужный шаг.
- `adjustCardGeometry()` — двигает ИТОГО и растит/сжимает container.
- `fillCard()` — заполняет тексты шейпов. Здесь же логика «10 000/мес» для подписочных строк.
- `checkPptxOverflow()` — watchdog, экспортируется и вызывается в KPPreview перед выгрузкой.
- В самом конце — `generateKPPptx()` — оркестрация: загрузка двух шаблонов, патч XML, перепаковка zip, скачивание blob.

Если хочешь понять структуру `.pptx`-XML — открой `public/templates/commercial_template.pptx` в архиваторе (это zip). Внутри `ppt/slides/slide1.xml` — то, что мы патчим.

### `src/lib/catalog.ts` (~580 LOC)

Просто данные. Прочти заголовки секций и одну запись из каждой:

- `tablets[]`, `mounts[]`, `peripherals[]`, `kioskKits[]`, `posEquipment[]` — встроенный fallback-каталог.
- `services[]` — внедрение (20 000 ₽), контент (1 200 ₽), BONDA BI.
- `innoLicenses[]` — цены лицензий ИННО.
- `findirTariffs[]` — тарифы ФинДир со ступенями цены по числу локаций.
- `periodMultiplier{}` — `month` / `quarter` / `half_year` / `year` → месяцы. Используется везде.
- `getProductById()`, `getLicensePrice()`, `getFindirPrice()` — lookup-хелперы.

### `src/app/page.tsx` (~1150 LOC)

Самый громоздкий, но прямолинейный. Не пытайся прочесть целиком — пройдись по структуре:

- В начале — `defaultForm`, `fallbackCatalog`, `Home` (главный компонент).
- `update()` — основной мутатор state (внутри — авто-сбросы зависимых полей при смене company/license_type).
- `handleGenerate()` — обогащение формы данными из каталога (`_kiosk_name`, `_kiosk_price`, `_kiosk_options_data`) и вызов `calculateKP`.
- Дальше JSX формы (огромный if-else по `license_type`).
- В конце — утилитарные функции (`fetchGoogleSheetProducts`, `parseRowToProduct`, `parseCSV`, `GoogleSyncButton`, `NumberInput`, `RadioCard`, `Section`).

### `src/components/KPPreview.tsx` (~600 LOC)

Превью на втором шаге. Прочти от `export function KPPreview` до конца:

- `sections` state — копия `kp.sections`, мутируется локально.
- `recalcItem` / `recalcSection` / `monthlyTotal` (computed).
- `undoLastRemoval`, `Removal` snapshot — undo-toast (Phase 1).
- `replaceWithProduct`, `updateField`, `removeItem`, `addItem`, `updateName` — мутаторы.
- `handleDownloadPPTX` — вызывает `checkPptxOverflow` → `generateKPPptx` → скачивание.
- В JSX — таблица позиций, undo-toast внизу.

## Часто возникающие вопросы

**Где править цены лицензий?**
`src/lib/calculator.ts → innoLicPrices` (внутри `calculateKP`). Дублируется в `src/app/page.tsx` (RadioCard labels) и в `src/lib/catalog.ts → innoLicenses`. Это известный долг H22 в аудите — три источника правды должны быть один. Пока правишь — синхронизируй все три.

**Как добавить новую модель планшета / киоска?**
Добавь в Google Sheet → нажми «Синхронизировать» в UI. Если хочешь добавить в встроенный fallback — `src/lib/catalog.ts → tablets[]` / `kioskKits[]`.

**Шаблон .pptx сломался / нужно изменить дизайн КП-слайда?**
Открой `public/templates/commercial_template.pptx` в PowerPoint. Меняй визуал. **Но не трогай shape names** (`Text 16`, `Shape 5` и т.д.) — `generatePptx.ts` ищет их по имени. Если переименуешь — код сломается без предупреждения.

**Менеджеры жалуются на разные числа в превью и в .pptx.**
Это симптом класса багов «preview ≠ pptx» (P0-1 / P0-2 в аудите). Проверь:
1. Запусти `checkPptxOverflow(currentKP)` мысленно — все секции в `PPTX_TEMPLATE_LIMITS`?
2. Все секции имеют title из списка `Оборудование / Лицензии и подписки / Услуги`?
3. `LineItem.months` корректно проставлен для подписочных строк?
Если всё да, а расхождение есть — это новый класс бага, открывай тикет.

**Где смотреть прод-логи?**
Vercel Dashboard → Logs. Но в нашем случае всё происходит client-side, поэтому в логах Vercel будет минимум — серверной обработки запросов почти нет.
